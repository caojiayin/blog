文字组织能力一直是我的弱项，想到写XX文档就头如牛。朋友们告诉我说多写blog、多看历史书就能提高这方面的能力，对于写blog还好吧，如果抱到一本厚厚的历史书籍，想想一下就头皮发麻，两眼泛白。

最近比较清闲，看到“傅二代”同志的github博客，并且了解了一下后，发现这玩意还挺好玩的。所以就搭建了这个blog。

还是得感谢“傅二代”同志的帮助和阮一峰的这篇“搭建一个免费的，无限流量的Blog”文章。

注意的是：在创建github的blog分支时一定要取名为gh-pages不然是无法访问，会出现404错误。可以用此命令检查和创建该分支$ git checkout --orphan gh-pages

安装好git或msysGit软件后(下载)。 如果你本机项目不在此软件下，如果用cd命令指定项目目录

常用命令查当前git状态，是否有文件未提交等(这是我删除本地文件后，还未提交时的状态)

平时用到的命令

	$ git add . //将当前工作目录中更改或者新增的文件加入到Git的索引中
	$ git commit -m '注释' //提交当前工作目录的修改内容
	$ git push origin gh-pages //将本地commit的代码更新到远程版本库中

ps:记住上面这些命令那么你就可以发布和管理你的blog了，是不是很简单也很好玩。

删除远程文件

删除远程文件时，最好先pull一下，将服务器端文件下载到本地项目内，在用rm命令删除，然后在commit一次，最后push一下就OK.

	$ git pull https://github.com/username/gh-pages.git //从服务器的仓库中获取代码，和本地代码合并
	$ git rm test.txt
	$ git commit -m 'delete files'
	$ git push origin gh-pages
